# 答辩演示脚本

> 建议演示时长：8-10 分钟

---

## 开场（30秒）

> 各位老师好，我做的毕业设计是"雪峰山智慧文旅系统"。系统包含三个端：
> - 微信小程序（游客端）
> - Vue 管理后台（管理员端）
> - Spring Boot 后端服务
>
> 核心亮点是 AI 智能导览、Redis 高并发防超卖、以及数据可视化大屏。

---

## 第一幕：小程序端（2分钟）

### 1. 首页浏览
- 打开小程序 → 展示热门推荐和景区列表
- "数据来自后端 API，热门推荐按浏览量排序取前3"

### 2. 景区详情 + AI 导览
- 点击一个景区 → 展示沉浸式封面、景区概况、核心景点
- 点击右下角 AI 悬浮按钮 → **现场演示 AI 对话**
  - 问："雪峰山有什么好玩的？"
  - 追问："门票多少钱？"（展示多轮对话能力）
- "AI 接入了 DeepSeek 大模型，支持上下文记忆"

### 3. 预约流程
- 点"立即预约门票" → 弹出登录框 → 输入手机号
- 选择时段 → 确认预约 → 生成二维码
- "预约使用 Redis 原子扣减库存，防止超卖"

### 4. 个人中心
- 首页点"个人中心" → 展示用户信息
- 点"我的预约" → 展示预约记录和入园二维码

---

## 第二幕：管理后台（2分钟）

### 1. 登录
- 打开管理后台 → 输入 admin / 123456
- "密码使用 BCrypt 加密存储，JWT 做身份认证"

### 2. 数据大屏
- 展示四个指标卡片（今日预约、累计接待、在线景区、核销率）
- 展示预约趋势折线图（7天/30天切换）
- 展示景区热度饼图
- "所有数据来自真实数据库查询，不是硬编码"

### 3. 核销演示
- 切换到核销中心 → 输入刚才预约的订单号
- 点"立即核销" → 显示"核销成功"
- "核销接口有并发安全保护，防止重复核销"

### 4. AI 助手
- 切换到 AI 助手 → 展示多轮对话
- "管理后台的 AI 也支持上下文，方便管理员了解景区信息"

---

## 第三幕：技术亮点（2分钟）

> 接下来介绍几个技术亮点：

### 1. Redis 高并发防超卖
- "预约接口使用 Redis DECREMENT 原子操作扣减库存"
- "扣减成负数时立即回滚，保证不超卖"
- "数据库写入失败时自动回滚 Redis 库存"

### 2. 安全体系
- "管理员密码 BCrypt 加密，JWT 令牌认证"
- "基于注解的角色权限控制（@AdminRequired）"
- "全局异常处理覆盖参数校验、JSON 格式、权限不足等场景"

### 3. 代码质量
- "AOP 统一记录请求日志（URL、IP、参数、耗时）"
- "Knife4j 自动生成 API 文档，支持在线调试"
- "前端表单校验 + 后端参数校验双重保护"

---

## 第四幕：数据库设计（1分钟）

> 简要介绍核心数据表：
> - `scenic_area` — 景区信息
> - `scenic_spot` — 子景点
> - `time_slot` — 预约时段
> - `reserve_record` — 预约记录
> - `tourist_user` — 游客用户
> - `sys_admin` — 管理员

## 收尾（30秒）

> 以上就是雪峰山智慧文旅系统的全部演示。
> 系统采用前后端分离架构，小程序端面向游客，管理后台面向运营人员。

## 可能被问到的问题

| 问题 | 参考回答 |
|------|---------|
| AI 用的什么模型？ | DeepSeek，通过 OpenAI 兼容协议调用，支持多轮对话 |
| 怎么防止超卖？ | Redis 原子扣减 + 负数回滚 + 数据库写入失败补偿 |
| 密码怎么存储的？ | BCrypt 加密，每次生成不同的盐值，不可逆 |
| 前后端怎么通信？ | RESTful API + JWT 令牌认证，CORS 跨域配置 |
| 并发能支持多少？ | Redis 单机 10 万+ QPS，瓶颈在数据库，当前配置支持百级并发 |
| 为什么用 Redis 不直接查数据库？ | 高并发场景下数据库扛不住，Redis 内存操作快 100 倍 |
